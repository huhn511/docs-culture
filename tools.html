Tools

This is a list of tools, which you can use to support your documentation workflows.

GitHub Actions

GitHub actions is a nice tool for automate things.

With this workflow running on GitHub Action, the documentation automaticly builds and deploy itself.

GitHub Pages

GitHub Pages allows you to host static content, like a website - and it's free!

mdBook

mdBook is a command line tool and Rust crate to create books using Markdown (as by the CommonMark specification) files. It's very similar to Gitbook but written in Rust.

Why mdBook?

incluce files

Perfect tool for the DRYprinciple. With the following syntax, you can include files into your book:

{{#include file.rs}}

Example (book.toml):

[book]
authors = ["huhn511"]
language = "en"
multilingual = false
src = "src"
title = "Documentation Culture"

mdBook multiverse

Where each project has a own documentation page build with mdbook, it's very distributed and hard to find all the pages of your project. If you want to combine multiple projects to one documentation, you also can use mdBook!

See this example and the code repository, which loads all projects with gitsubmodules. Here is a Github Action for auto deployment.
